# Godot Game Club Project Brief
## Build a Tiny 2D Game

### The challenge
Build a **small 2D game** in Godot.

Your game must have:
- a player that can move
- one small level or room
- a clear goal
- at least one obstacle or hazard
- a win message or win screen
- a way to restart and play again

### You can choose your own theme
Your game can be about anything you like, for example:
- a treasure cave
- a haunted house
- a school escape
- a robot factory
- a wizard maze
- a jungle rescue
- a silly dream world
- Ledbury!

## The rules
### You must include
- **one player**
- **one level**
- **one goal**
- **one obstacle**
- **one win state**
- **one restart option**

### You may optionally add any of these...
- sound effects
- music
- animations
- collectibles
- dialogue
- extra obstacles
- a title screen
- a score

### Avoid 🙅
- multiplayer
- giant open worlds
- lots of levels
- complicated combat
- inventories with many items
- character customisers
- building games "like Minecraft" or "like Roblox"

Start small. Make **one thing that works**.

# Weekly Plan

## Week 1: Build the space
### Goal
Create your project and make a small level.

### You should try to:
- make a new Godot project
- add a background, floor, walls, or tiles
- import sprites or a tileset
- save your level as a scene

### By the end of the session
You should have **a world to walk around in later**.

---

## Week 2: Make the player move
### Goal
Create a player character with movement.

### You should try to:
- add a player scene
- use keyboard input to move
- add collision so the player cannot walk through walls
- choose a player sprite

### By the end of the session
You should have **a playable character that can move around your level**.

---

## Week 3: Add the goal
### Goal
Create the object or place the player is trying to reach.

### Example goals
- treasure chest
- exit door
- magic crystal
- button
- animal to rescue
- lost item to collect

### You should try to:
- add a goal object
- detect when the player reaches or touches it
- show a message such as **"You Win!"**

### By the end of the session
You should have **a game that can be won**.

---

## Week 4: Add one obstacle
### Goal
Create one thing that makes the game harder.

### Example obstacles
- spikes
- enemy
- moving hazard
- locked door
- slippery floor
- timer
- patrolling robot
- fake goal

### You should try to:
- add one obstacle
- make it affect the player
- reset the player, block the player, or slow them down

### By the end of the session
You should have **challenge in your game**.

---

## Week 5: Polish
### Goal
Make the game feel better.

### You could add:
- animation
- sound effects
- music
- title screen
- restart button
- better art
- labels or UI
- particles
- a game over screen

### By the end of the session
You should have **a more fun and complete game**.

---

## Week 6: Showcase and improve
### Goal
Play each other's games and make one improvement.

### You should:
- let someone else play your game
- watch what they do
- listen to feedback
- improve one thing

### Good feedback questions
- Was it clear what to do?
- Was it fun?
- Was anything confusing?
- Was it too easy or too hard?
- What was your favourite part?

---

# Suggested Minimum Build
If you are stuck, make this:

- one room
- one player
- one exit door
- one hazard
- one win message
- one restart button

That is enough for a complete game.
